SkillHub — how to use

Two paths, depending on your role: receive the skills (most people) or create and publish them. Pick yours below.

everyone

I'm a consumer

I want to use the skills in my Claude and get updates.

see the steps ↓ a few

I'm an author

I want to create new skills and publish them for the team.

see the steps ↓

Consumer — receive and update skills

In the Claude desktop app. You install once; after that you just refresh when told a new version shipped.

Prerequisite · once

Get access to the repository

Ask to be added as a collaborator on rafaelnetovec/claude-skills and accept the invite (GitHub emails it). Without it, Claude can't see the private repository.

Install · once

Open plugins

In Claude (desktop app): Settings → Plugins.

Add the marketplace

Click Add (or the + in the Directory) → add from GitHub → enter rafaelnetovec/claude-skills.

Authorize GitHub

If prompted, connect the GitHub account that has access to the repository.

Install “Vec skills”

Open the plugin and install it. Skills appear as /vec-example, /text-reviewer, etc.

Update · whenever something new ships

Open the marketplace menu

Plugins → Directory → "Personal" tab → on claude-skills, click the … menu.

Check for updates

Click “Check for updates”. Then reopen the skill (or the app). New skills and new versions appear.

!
Gotcha: reinstalling the plugin does NOT update it (it uses a cache). Updates only arrive via “Check for updates” on the marketplace menu. The “Sync automatically” toggle exists, but today it depends on the GitHub App being granted access / moving the repo to an organization.

Author — create and publish skills

You use the skill CLI, which handles structure, versioning, and publishing. GitHub Actions validates and creates releases automatically.

Prerequisites · once

WRITE access to the repo

Be a collaborator with Write permission on rafaelnetovec/claude-skills (consumers only need read; authors need write to publish).

Git and Python 3.10+

Confirm with python --version.

Set up your machine · once

PowerShell
# 1. clone the repo (Git opens a browser to sign in the first time)
git clone https://github.com/rafaelnetovec/claude-skills.git
cd claude-skills

# 2. allow running .ps1 scripts in this window
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

# 3. run the bootstrap — checks git/python, installs the CLI, verifies access
.\scripts\bootstrap-autor.ps1

# (manual alternative to step 3: python -m pip install -e .)

Create and publish a skill

PowerShell
# 1. scaffold a new skill
skill create my-skill --owner growth --description "What the skill does."

# 2. edit skills/my-skill/SKILL.md and set the status in skill.yaml

# 3. validate
skill validate my-skill

# 4. bump the version (patch | minor | major)
skill bump my-skill minor

# 5. publish (CI creates the tag/release)
skill publish my-skill --no-release
!
Two GitHub accounts on one machine? If the push fails with 403 (“denied to OTHER-ACCOUNT”), point the remote at the right user: git remote set-url origin https://YOUR-USER@github.com/rafaelnetovec/claude-skills.git. Multiple authors? Work on a branch and open a Pull Request — CI validates before the merge. Or just ask Claude Code (in the repo) to publish for you.

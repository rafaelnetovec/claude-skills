Consumidor — receber e atualizar skills

No app desktop do Claude. Você faz a instalação uma vez; depois é só atualizar quando avisarem que saiu novidade.

Pré-requisito · uma vez

Ter acesso ao repositório

Peça para te adicionarem como colaborador em rafaelnetovec/claude-skills e aceite o convite (chega por e-mail do GitHub). Sem isso, o Claude não enxerga o repositório privado.

Instalar · uma vez

Abrir os plugins

No Claude (app desktop): Configurações → Plugins.

Adicionar o marketplace

Clique em Adicionar (ou no + do Diretório) → adicionar do GitHub → informe rafaelnetovec/claude-skills.

Autorizar o GitHub

Se pedir, conecte a conta do GitHub que tem acesso ao repositório.

Instalar “Vec skills”

Abra o plugin e instale. As skills aparecem como /exemplo-vec, /revisor-pt, etc.

Atualizar · sempre que sair novidade

Abrir o menu do marketplace

Plugins → Diretório → aba “Pessoal” → no claude-skills, clique no menu ….

Verificar atualizações

Clique em “Verificar atualizações”. Depois reabra a skill (ou o app). Skills novas e versões novas aparecem.

!
Pegadinha: reinstalar o plugin NÃO atualiza (ele usa cache). A atualização só vem pelo “Verificar atualizações” no menu do marketplace. O toggle “Sincronizar automaticamente” existe, mas hoje depende de liberação do GitHub App / migração para organização.

Autor — criar e publicar skills

Você usa a CLI skill, que cuida de estrutura, versão e publicação. O GitHub Actions valida e cria os releases sozinho.

Pré-requisitos · uma vez

Acesso de ESCRITA ao repo

Ser colaborador com permissão Write em rafaelnetovec/claude-skills (consumidor precisa só de leitura; autor precisa escrever para publicar).

Ter Git e Python 3.10+

Confirme com python --version.

Preparar a máquina · uma vez

PowerShell
# clonar o repositório
git clone https://github.com/rafaelnetovec/claude-skills.git
cd claude-skills

# instalar a CLI do SkillHub
python -m pip install -e .

# conferir (se "skill" não for encontrado, use: python -m skillhub.cli)
skill list

Criar e publicar uma skill

PowerShell
# 1. criar o esqueleto
skill create minha-skill --owner growth --description "O que a skill faz."

# 2. editar skills/minha-skill/SKILL.md e ajustar o status em skill.yaml

# 3. validar
skill validate minha-skill

# 4. subir a versão (patch | minor | major)
skill bump minha-skill minor

# 5. publicar (o CI cria a tag/release)
skill publish minha-skill --no-release
!
Duas contas GitHub na máquina? Se o push der 403 (“denied to OUTRA-CONTA”), aponte o remote para a conta certa: git remote set-url origin https://SEU-USUARIO@github.com/rafaelnetovec/claude-skills.git. Vários autores? Trabalhe em branch + Pull Request — o CI valida antes do merge.

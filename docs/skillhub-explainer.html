VEC · Claude Code

SkillHub

O jeito da VEC criar, versionar e distribuir as skills do Claude — sem copiar arquivo à mão, sem ninguém ficar em versão diferente.

As skills deixaram de ser prompts soltos no computador de cada um. Agora existe um repositório único como fonte da verdade: um desenvolvedor publica com um comando, e a skill chega automaticamente no Claude de todo mundo — validada e versionada.

Por que isso importa

O que muda na prática, para quem cria skills e para quem só usa.

Sempre atual

Igual a um app do celular

As skills se atualizam sozinhas quando o Claude abre. Ninguém instala nem baixa nada manualmente.

Qualidade

Toda skill é validada automaticamente antes de chegar ao time. Cada versão tem histórico e um responsável.

Por dentro — para desenvolvedores

As peças da engrenagem e a CLI que o autor usa. Pense nela como o npm, só que para skills.

~/claude-skills — skillhub
skills/<nome>/Fonte da verdade. Cada skill tem SKILL.md, skill.yaml, CHANGELOG.md e tests/.
skill.yamlA etiqueta da skill: nome, versão (semver), dono e status. É o que dá controle de qualidade.
registry.jsonO catálogo com todas as skills e versões. Gerado automaticamente.
CLI · skillO atalho do autor: criar, validar, versionar e publicar sem mexer em Git.
GitHub ActionsO “funcionário robô”: valida no PR, sincroniza o catálogo e cria release por skill.
Marketplace nativoO canal de distribuição do Claude Code. Cada pessoa cadastra uma vez; o resto é pull automático.

A CLI em um relance

skill create <nome>Cria o esqueleto de uma skill nova.
skill validateConfere estrutura e metadados (uma ou todas).
skill listLista as skills com versão, dono e status.
skill bump <nome> minorSobe a versão (1.0.0 → 1.1.0) e registra no CHANGELOG.
skill publish <nome>Valida → catálogo → commit/push → tag + release.

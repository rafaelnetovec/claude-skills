VEC · Claude Code

SkillHub

How VEC creates, versions, and distributes its Claude skills — no copying files by hand, no one stuck on a different version.

Skills are no longer loose prompts on each person's computer. There's now a single repository as the source of truth: a developer publishes with one command, and the skill reaches everyone's Claude automatically — validated and versioned.

Why it matters

What changes in practice — for those who create skills and for those who just use them.

Always current

Like a phone app

Skills update themselves when Claude opens. No one installs or downloads anything by hand.

Quality

Nothing ships broken

Every skill is validated automatically before it reaches the team. Each version has history and an owner.

No friction

One command publishes

Authors don't touch folders or Git. The machinery validates, versions, and distributes.

There are only two kinds of people

Most of the company is a consumer — and for them there's no work at all.

a few

Skill author

Those who create and maintain skills.

  • Writes the skill and publishes it via the skill CLI.
  • Sets version, owner, and status (draft → production).
  • Needs only basic Git — the tooling does the rest.
everyone

Consumer

Those who use the skills day to day.

  • Does nothing: skills appear and update on their own.
  • Subscribes to the distribution channel once.
  • Never needs to know Git or versions.

Under the hood — for developers

The moving parts and the CLI the author uses. Think of it as npm, but for skills.

~/claude-skills — skillhub
skills/<name>/Source of truth. Each skill has SKILL.md, skill.yaml, CHANGELOG.md, and tests/.
skill.yamlThe skill's label: name, version (semver), owner, and status. This is what gives quality control.
registry.jsonThe catalog of every skill and version. Generated automatically.
CLI · skillThe author's shortcut: create, validate, version, and publish without touching Git.
GitHub ActionsThe “robot worker”: validates on PR, syncs the catalog, and creates a release per skill.
Native marketplaceClaude's distribution channel. Each person subscribes once; the rest is automatic pull.

The CLI at a glance

skill create <name>Scaffold a new skill.
skill validateCheck structure and metadata (one or all).
skill listList skills with version, owner, and status.
skill bump <name> minorRaise the version (1.0.0 → 1.1.0) and record it in the CHANGELOG.
skill publish <name>Validate → catalog → commit/push → tag + release.

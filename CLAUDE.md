# CLAUDE.md — working in this repo

This repository is **SkillHub**: the single source of truth for VEC's shared Claude
skills. It ships a CLI (`skill`), a generated registry, a generated plugin consumed by
the Claude marketplace, and CI that validates and releases.

When the user asks you to **create, change, version, or publish a skill**, do it for
them by running the CLI and pushing — don't just tell them the commands. Follow the
workflow below exactly.

## Layout

- `skills/<name>/` — source of truth for each skill:
  - `SKILL.md` — the skill itself (YAML frontmatter with `name` + `description`; the
    `description` is what makes the skill trigger, so write it in **English** with clear
    trigger phrases).
  - `skill.yaml` — metadata: `name`, `version` (semver), `owner`, `status`
    (`draft` | `beta` | `production` | `deprecated`), `description`, optional `tags`.
  - `CHANGELOG.md`, `tests/`.
- `registry/registry.json` — generated catalog. Never edit by hand.
- `plugins/vec-skills/` — generated plugin the marketplace serves. Never edit by hand.
- `.claude-plugin/marketplace.json` — generated. Never edit by hand.
- `skillhub/` — the CLI source (Python/Typer).

> The CLI may be on PATH as `skill`; if not, use `python -m skillhub.cli` instead.

## Publish workflow (what to run)

```
skill create <name> --owner <team> --description "<what it does>"   # new skill
# edit skills/<name>/SKILL.md (write real content; set status in skill.yaml)
skill validate <name>                 # must pass
skill bump <name> <patch|minor|major> # whenever content changed
skill publish <name> --no-release     # validate + regenerate + commit + push
```

- Use `--no-release`: the GitHub Actions workflow creates the tag and release (it does
  not depend on `gh` being authenticated locally).
- After the push, CI validates, regenerates `registry.json` + the plugin, and creates a
  per-skill tag/release (`<name>-v<version>`).
- Always `git pull` before starting (multiple authors).

## Rules

- Only touch `skills/`. `registry/`, `plugins/`, and `.claude-plugin/` are generated by
  `skill registry` / `skill build` — regenerate them, never hand-edit.
- Every content change needs a version bump (`skill bump`); the desktop app detects
  updates via the plugin `version`, which is derived from the sum of skill versions.
- If you ever **remove/rename** skills and the aggregate version would drop below what
  was already published, bump `PLUGIN_VERSION_EPOCH` in `skillhub/core.py` so the plugin
  version keeps increasing (otherwise clients won't see the update).
- Skill descriptions and user-facing skill content are in **English** (the audience is
  English-speaking).

## Gotchas

- **Two GitHub accounts on one machine:** if `git push` fails with 403 "denied to
  OTHER-ACCOUNT", point the remote at the right user:
  `git remote set-url origin https://<user>@github.com/rafaelnetovec/claude-skills.git`.
- The repo is **private**: consumers need read access (collaborator/org) and authors
  need write access.
